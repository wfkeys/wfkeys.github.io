webpack-dev-server

webpack-dev-server 为你提供了一个简单的 web server，并且具有 live reloading(实时重新加载) 功能。通过这个工具可以实时监视代码改变，自动打包，打包完自动刷新浏览器。这里自动打包的是server上面的缓存文件，contentBase: ‘./dist’,这个./dist是虚拟目录，这样可以做到快速调试写功能，最后还是要进行一次webpack命令打包，才可以使本地的bundle.js进行修改。

babel-loader

此 package 允许你使用 Babel 和 webpack 转译 JavaScript 文件,将ES6转成ES5

安装

配置babel-polyfill来处理低版本API的兼容问题
配置transform-runtime解决打包代码重复问题

在打包过程中，babel会给一些包提供工具函数，有多少个包就提供多少个函数，就会导致里面有些工具函数是重复出现多个的，导致打包体积过大，babel提供了一个babel-transform-runtime来解决这个问题

安装和配置

你必须执行 npm install -D @babel/plugin-transform-runtime 来把它包含到你的项目中，然后使用 npm install @babel/runtime 把 @babel/runtime 安装为一个依赖。

babel打包速度慢问题

cacheDirectory：默认值为 false。当有设置时，指定的目录将用来缓存 loader 的执行结果。之后的 webpack 构建，将会尝试读取缓存，来避免在每次执行时，可能产生的、高性能消耗的 Babel 重新编译过程。默认它会将缓存在node_modules中，多出一个.cache目录
